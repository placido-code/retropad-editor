Config hint

RetroArch overlays are built from a collection of images and a text configuration file.

The config file (.cfg) describes:

  • The hitbox for each input event, i.e. "size" of the button.
  • Which images (.png) should be shown over the input elements.

More information: Libretro docs.

To use default "flat" image set: download and unpack archive, place config in folder with images.

Additional porpeties hint

The | character allows multiple commands to be chained together.

E.g. this will trigger both left and up at same time. 

left|up

Keyboard keys are an exception, a button can only trigger one key.

Allows an overlay_next button to jump directly to a specific overlay by name.

The alpha value of the button will be multiplied by a factor. 

alpha_mod = 2.0 # Alpha is multiplied 2x when pressed.

It's possible to set a default across all buttons for an overlay using the alpha_mod overlay property.

Makes the hitboxes larger while the button is pressed. 

range_mod = 1.5 # The hitbox will be 50% larger when pressed.

It's possible to set a default across all buttons for an overlay using the range_mod overlay property.

Used to configure where saturation kicks in for a analog_left or analog_right.

analog_left and analog_right translate to analog sticks. These buttons must have hitbox type radial.

E.g. a value of 0.75 means the 75% inner part contains the entire analog range. Outside the 75% it will be fully saturated.

Enables an overlay to follow the finger movement within its bounding area.

Adjusts a button's hitbox without affecting its image, input center, or analog saturation.

Use reach_x/y to multiply hitbox width/height symmetrically.

Use reach_up/down/left/right to multiply hitbox range in that direction.

E.g. this creates a d-pad and extends its hitbox vertically by 60%, and left by 40%. 

# dpad_area
reach_y = 1.6 # Multiply hitbox height
reach_left = 1.4 # Multiply hitbox width on left side
Extended hitbox Default hitbox

Setting reach_x or reach_y to 0 removes the hitbox. The button then can't be pressed, but it will still light up if another button creates the same input. This is useful for animating a dpad_area or abxy_area.

Used to avoid unwanted button overlap.

An exclusive button blocks other buttons pressed by the same finger.

A range_mod_exclusive button blocks with higher priority, but only when range_mod is in effect, i.e. one poll after it is pressed.

E.g. this overlaps an analog stick and a d-pad. The analog is exclusive, so initially it owns the overlapped hitbox area. Both buttons are range_mod_exclusive, so the one controlled first owns its range_mod extended hitbox area. 

# dpad_area
reach_x = 2.08
reach_up = 1.62
reach_down = 1.96
range_mod = 1.2
range_mod_exclusive = true # Give dpad_area priority if controlled first

# analog_left
exclusive = true # Give analog_left priority on first touch
range_mod = 2.75
range_mod_exclusive = true # Give analog_left priority if controlled first
range_mod_exclusive analog
range_mod_exclusive dpad

Used to redefine a dpad_area or abxy_area inputs.

dpad_area and abxy_area are 8-way areas whose input events are based on the direction pressed, i.e. angle from hitbox center. An 8-way area can be used as-is or redefined.

E.g. this creates a d-pad consisting of analog directions, and a face button diamond consisting of A, B, R, L. 

# dpad_area
up = r_y_minus
down = r_y_plus
left = r_x_minus
right = r_x_plus

# abxy_area
up = r
left = l
